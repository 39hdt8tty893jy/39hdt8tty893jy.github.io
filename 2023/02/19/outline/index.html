轮廓线 Outline

轮廓线 Outline

  轮廓线在传统游戏中很常见，通常表示指向或选定的单位，标识被建筑或地形遮挡的单位。而卡通渲染中，不仅要勾勒外轮廓，还要在内部勾线表现其结构。对于赛璐璐风格的作品，线条是非常重要的，既要求粗细虚实，又要手绘、水墨等各种风格化，在黑白漫画中要求更甚。所以轮廓线在 NPR 中有着不可忽略的重要性，而且也应该由美术主导，给与美术更多的线条控制。

  一般来讲我们所说的轮廓线是指外轮廓，如选定一个单位会勾勒出该单位的外轮廓线，或者视野被遮挡时用外轮廓线来表明这里还存在一个单位，如 2D 的《辐射》《英雄无敌》、3D 的《神界原罪》《暗黑3》等等。除了外轮廓，在 NPR 中有大量的同色色块需要用线条来表达内部的结构和关系，详见各种赛璐璐的动漫作品。《Real-Time Rendering 4th Edition》介绍了卡通渲染中 5 种轮廓边缘的类型：

  • boundary edge(B)：边缘只为一个三角形占用，没有体积，像是一张平铺纸，只能见到纸的一面。
  • crease edge(C)：边缘为两个三角形共用的折叠硬边缘，小于一个预设角度，如立方体的边。
  • contour edge：边缘为两个三角形共用，且两三角形相对于某个向量是不同方向的。通常以视角向量来检测面片是否面向视角或背向视角。
  • material edge(M)：不同材质着色的边缘。
  • silhouette edge(S)：外轮廓边缘，它的边缘类型和 contour edge 一样，只是其为外轮廓时特指，所以只能算 contour edge 一个子集。有时 boundary edge 也会当作外轮廓边缘，但其边只由一个三角形占用且没有体积表达，所以和 silhouette edge 有差异。
    图片截取自《Real-Time Rendering 4th Edition》

  注意在不同角度下这几类边会有不同表现，若以视角向量检测，crease edge 中两个面片都同时出现则为 crease edge；否则为 contour edge。图中 boundary edge 有时也作外轮廓边缘，但因为没有厚度或体积，所以还是有区分的。很多地方都没说明为什么要分出这几种轮廓线（Pencil+ 4 和 blender 的文档中还有其他的 edges ）。主要是和动漫的美术风格相关。表现外轮廓和结构转折明显的线条粗实，而内部轮廓和结构转折缓和的线条细虚，线条不能都都长一个样，甚至朝光面和被光面的轮廓线都有虚实。所以 NPR 渲染中我们主要针对 contour edge 进行勾线，表现外轮廓和内部结构体积。这里并不是说其他的边缘不用勾线，可再根据需求选择，比如有时内部的 crease edge 也需要勾。

《Real-Time Rendering 4th Edition》总结了如下几种描边的方法：

  • Shading Normal Contour Edges：

      1 个 Pass 很简单，如果顶点法线和视角向量内积为 0，即相互垂直，那么该顶点处在边缘上。因为很多情况边缘顶点不一定完全垂直视角向量，所以给定某阈值来粗略地判断。上图看出很明显的问题，立方体这种没有平滑过渡的“面”在特定角度很容易被直接渲染成“边”，而且就算是曲面表现也不尽如意，很难完美匹配边缘法线和视角向量垂直，轮廓线会显得有些潦草。我还基于像素进行描边，基于顶点效果更差。图中就只有球体的轮廓线较为均匀，能得到较好的描边效果。因为后面演示示例太多了，就不贴代码了。

  • Procedural Geometry Silhouetting：
      用 2 个 Pass，先渲染模型正面，再渲染背面当作轮廓线。因为通常背面肯定被正面遮挡，所以下面几种方式能让背面的边缘渲染出来：

    1. 背面偏朝相机前移，按深度 z 偏移。缺点是线条的粗细和两面片的夹角有关，粗细不一致。
      图片截取自《Real-Time Rendering 4th Edition》

      首先是背面朝相机前移，除了线条粗细不均匀外，还有两个极端很容易出现：背面和正面夹角很小，一丢丢偏移就完全遮挡正面；平滑曲面需要更大偏移才能观察到轮廓线。尤其模型中两种结构都很多时，下图中耳朵比较薄，而头比较圆，所以耳朵背面都覆盖正面了，但脑袋的轮廓线只有薄薄几个像素。书中 Figure 15.9 也有展示。我查到 Unity ShaderLab command: Offset 中主要是解决 z-fighting 和 shadow acne 这种深度精度的问题，对轮廓线帮助不大。

  1. 背面直接直接在其平面内展开放大，优点是边上的轮廓是粗细均匀的，缺点是，但角上度数越小，轮廓越突出。

    图片截取自《Real-Time Rendering 4th Edition》

  2. 背面直接在其平面内展开，但三条边只是向外平移不放大，最后再把顶点连接起来，这样可以避免上述小角度时突出的轮廓线。《波斯王子4》和 PS3 《爆炸头武士》就是用的这种方式。

    右侧是一个 Quad 的背面，只进行勾边

      和之前一种方式都是在面片平面展开，我选了复杂点的后者演示。因为要用到几何着色器，还在面剔除之前，所以兼容性和性能会差，而且角处没有了尖。既然用到了几何着色器就合并了 1 个 pass，对于背面也只勾边，稍稍扳回点性能。

  3. 除了背面在其平面内扩展，还可以让顶点按其法线扩展，这样边缘自然会扩展出来。缺点是相同的顶点在不同面片按照各自的法线移动，所以法线夹角过大时，轮廓线连接处会断开。而解决这个问题需要按照其平均法线扩展，所以需要预先制作顶点的平均法线。

  从上图可以看出，具有弧度的边缘能取得不错的效果。但有两个小问题：一是拉近观察，立方体这种硬边缘的顶点处轮廓线断开；二是背面顶点按法线偏移后，因为视角原因，尤其是背面法线朝屏幕里时，会看不到轮廓线。这其实是一个问题，可以预先计算顶点的平均法线，这样各顶点按平均法线偏移后就不会出现断裂，也能保证背面扩展到正面的外侧。就像 Unity 导入模型时考虑平滑角度计算的法线，只要把角度设置大于 90° 就不会出现立方体轮廓线断开的情况。但这时立方体的面渲染不正确，所以需要额外的存储通道来保存平均法线。而实际项目中美术可能会修改顶点法线，所以平均法线的计算还需要额外处理。

立方体角上的轮廓线断开

背面按法线偏移始终没法观察

  上述 4 种都是顶点的偏移做轮廓线，无论是 z 方向，三角面片扩展放大方向，还是法线方向，那么必然存在两种通病，即偏移后的顶点和其他顶点产生深度上的问题，甚至 z-fighting。只能减少顶点偏移量，来缓和这个问题。二是无论哪种偏移，勾勒连续轮廓的多个背面，可能和正面夹角不均匀，或者各顶点的法线方向差异比较大，导致顶点偏移也不均匀。这样一条完整的边出现不规则的粗细现象。可以想象一个半球体，弧面正对相机，所以理想的轮廓线应该是一个均匀平滑的圆。但如果背面边缘处是坑坑洼洼的随机凹凸面，那么正面的轮廓线就很不美观了。这个可以在视图空间中按单位法线的 XY 偏移来解决。还有一个小问题，透视导致的远处顶点偏移“变少”，所以轮廓线也近大远小，其实这很符合近实远虚的特点。但如果有需求要线条等宽，就像 Unity 编辑器中选定几何体时的轮廓线不论远近都是一样的粗细。那么需要考虑 aspect ratio 再按法线的 XY 偏移即可。最后可以预先给出顶点偏移系数，不同的系数顶点偏移程度不一，也可以缓和上述通病，也提供了更多的线条控制。

简单拉个模型为两个圆锥构成，正面是标准的，背面顶部是偏斜的

调整为平均法线和计算 aspect 后，断裂不均等问题修复

  • Edge Detection by Image Processing：
      基于图像的边界检测，每本图形学教材上都有，就不再赘述了。检测有两部分：一是场景至少已经完成几何阶段渲染，像是 GBuffer 中保存了法线、深度、Object/Material ID或者其他数据。二是根据这些信息结合各种图像处理的卷积模板，判断是否为边。例如根据各种 ID 等等可以完美找到 boundary 和 silhouettes；像许多 contour edge 可以根据深度和法线找到。但也不是万无一失的，像是桌子上堆叠一张张纸，这种 boundary edge 深度和法线都没法区分，而且有时还会在不该生成轮廓线的生成。所以给出深度和法线的范围来决定。《无主之地》就采用了基于图像的边缘检测，并做了 MASK 蒙版剔除不需要描边的物体，如地上的草。

后处理中利用深度和法线做边缘检测

拉近距离后错误消失

  上图中可以看出利用深度和法线可以得到较好的结果，但仍然会产生一些小问题，比如右上角猴子的耳朵出现了较大的边缘着色。因为透视远处的图像变小，于是法线差异就变大导致了这种情况，拉近视野就正常了。（遗留问题：针对这种情况《无主之地》通过法线深度图的梯度来处理，而不是只给与一个阈值来判断。还没看文章，个人猜测可能是根据深度来调整法线判断的阈值。）《二之国2》在基于图像处理的描边上非常成功，还提供了美术对内部勾线的控制，虽然成本有点高，详细的参考《二之国2》的帧分析。

  • Geometric Contour Edge Detection：
      上述几种方法对于描边的风格化限制比较大。而基于几何图形的边缘检测可以精确地找出轮廓边缘。比如一条 contour edge 可以根据相邻的两个三角面片法线来判断：

    (n0⋅v)(n1⋅v)<0(n_0 \cdot v)(n_1 \cdot v) < 0 (n0​⋅v)(n1​⋅v)<0

      和前面挤出背面来模拟边不同，该方式能通过几何面精确找出边，构建风格化的轮廓线，几何着色器中能很方便实现。优化方式：相邻三角形在同一平面或凹陷的边缘，这些都不同做检测。同时如果相机和物体只有极小的偏移那么仍然可以重用之前的边缘。（遗留问题：建立风格化描边库，像 blender freestyle 和 pencil+ 中那样。）

  • 在 UV 贴图中勾线：

    观察人物内部的勾线

      早在 17 年前《代号13》就能发现，有些勾线直接画在贴图上，如衣服上的褶皱。正面静止看着还行，但只要移动或者旋转后就不那么完美了，一是死板不生动，二是锯齿，不过近 20 年前以爽快为主的游戏，那种锯齿还是可以接受的。否则要增加模型面数，用更复杂的结构表达出细节。还有如上图中，三角肌/胸大肌/肱二头肌三头肌上和一些衣物内部的勾线，这些区域在模型上过渡很平滑的，但添加勾线能明显增强肌肉的主观表达和衣物的结构细节。所以这种方式的勾线只能在少数特殊的地方由美术来画，不是程序想当然根据规则来决定。很明显如同《代号13》，最直观的方式就是美术直接在贴图中勾线，但这样做最大的问题是锯齿。那么上更高分辨率纹理也可以缓解，但仅仅为了勾边又很浪费。所以为了解决锯齿有下面两种方式：

    • 本村线：大名鼎鼎的本村线几乎是完美的，可以契合手绘的笔触，在贴图中用横平竖直的方式勾线，避免了锯齿的产生，之所以用“几乎”是因为苦了美术调整 uv。另外本村线只是添加细节的勾线，完全由美术制作，不是所有的内部勾线都由本村线完成。为了减轻美术工作量，可以只在勾边处单独做 uv，这样对其他部分的贴图没啥影响，减少了美术的工作量，但会再增加面数和贴图，就像 Mesh 勾线。
    • SDF：直接用一张额外的 SDF 图存储描边，即能改善线条的锯齿，又能提供对线条较好的控制。缺点是：需要额外一张高精度 SDF 贴图开销，仅仅只勾几条线。

  • 其他的外轮廓描边：
      最早的游戏描边无外乎分为两个 pass，先渲染一个大一圈的，然后把正常的渲染嵌入进去，自然能得到外轮廓。学习 Stencil Buffer 模板缓冲通常的案例就是外轮廓描边。也有直接渲染到 RT 上，直接放大 RT 图像或高斯模糊，再正常渲染的。

  好了总结的差不多了，为了得到更好的描边质量，通常需要多种方式联合使用。另外线条还要尽可能给美术多的控制，像是基于图像处理的边缘检测要暴露出更多参数，需要将边缘强度、着色强度、ID（Mesh/Material）等等放置不同通道中渲染到一张甚至多张 RT 中，成本比较高。但如果对性能有要求，选择外轮廓 + 本村线也基本可行。
