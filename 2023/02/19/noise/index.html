噪声之一

游戏开发中常用的噪声算法之一

  噪声简单来讲就是随机数。通常在计算机中模拟现实画面时，会应用到噪声，让简单又机械又呆板的画面更有真实感，如平整的大地有了起伏，水面有了波浪，粒子运行也不会显得千篇一律，火焰、溶解等特效也变得合理。不同的噪声有不同的表现，下面主要介绍游戏中相关的图形噪声。

1. 白噪声

  最简单的噪声就是白噪声，老电视的雪花屏就是典型的白噪声。每个像素点都是完全独立的随机值，和其他像素并不相关。下面就是通常生成伪随机白噪声的算法模式，很方便扩展到 2D 或 3D。基本思路就是用正弦处理待输入数据，如时间、纹理 uv 和坐标等等，再用 frac 裁掉整数部分，只保留小数部分即可。调整随机种子即可变化新的噪声值，如果还想更随机点，可以考虑加入其他变量，如时间。

白噪声图像

  因为上面用到了三角函数，如果每帧都有大量噪声计算，且平台内置的 sin 或 cos 函数成本高，性能优化时可以考虑整型计算的 Jenkins Hash。但如果要随机的浮点数，还是要做乘除法。提前生成一组随机值数组，直接查表也是不错的办法。遗留问题：DX/OPENGL/VULKAN 等等验证三角函数和 Hash 的性能比较，只用验证 sin 即可。

2. Value Noise

  自然界中人感受到的白噪声主要来自声音，所以白噪声在游戏画面中几乎很少用到，除了上面老电视的雪花屏和模拟劣质老胶片或电影布满噪点之类。对比每一个采样点都是独立随机的白噪声，其实现实中多数噪声还是有“规律”的，就是噪点与噪点之间有一定的过渡渐变。Value Noise 就是这类最简单的噪声。我们可以把待处理数据分成多个相同的 Cell。1D 就是分割成线段，2D 分割为正方形，3D 分割正方体。然后利用上面白噪声的伪随机（固定点的随机值是固定的），确定顶点的随机值。最后各个噪声点根据周围顶点的随机值做线性插值计算，即可得到有一定过渡渐变的噪声。我之所以用 Cell，因为虽然多数称为 grid 或晶格，又容易和后面细胞噪声混淆。但首先不一定就是 grid，就像 unity 中 2D Tilemap 就区分了 Grid、Isometric 和 Hexagon 类型，只要能铺满一个空间且中间没有空隙的都行，如等边三角形、正六边形，三维中如四面体等等。所以既然借鉴了晶体结构的概念，我还是喜欢用能反应晶格对称特性的最小几何单元——晶胞 Cell。（小思考：为什么是这些基础的图形？其他图形不可以吗？一是要铺满整个空间，如正五边形就做不到平面密铺；二是图形要单一简单方便计算，总不能用 15 种特殊的五边形去填充一个平面吧；三是要分布均匀，如直接把 Grid 按对角线分割一次，即两个等腰直角三角形，噪声点按照三角形顶点计算，那么对角线上的噪声过渡会出现明显的生硬。）

  Value Noise 明显的问题就是 Cell 的顶点是径向对称，边界是轴对称的，Cell 越多越能看出来，且线性过渡很生硬，一阶导数是个常量，在边缘处是不可导的。于是想办法做个平滑点的过渡，有 2 种常用计算插值的多项式及其导数：

y=3x2−2x3⇒y′=6x(1−x)y=6x5−15x4+10x3⇒y′=30x2(x−1)2\begin{align*} y & = 3x^2 - 2x^3 \Rightarrow y' = 6x(1-x) \\ y & = 6x^5 - 15x^4 + 10x^3 \Rightarrow y' = 30x^2(x-1)^2 \end{align*}yy​=3x2−2x3⇒y′=6x(1−x)=6x5−15x4+10x3⇒y′=30x2(x−1)2​

  前者计算比后者简单，但后者高阶导数仍然可导，在 Cell 顶点和边界处过渡会更平滑，高维度过渡更自然。

3. Perlin Noise

  上面可以看出 Value Noise 还是不自然，虽然点与点之间有了一定过渡，但顶点和边缘处有轴对称的特点，能较明显地看出晶格的分割。因为 Value Noise 的顶点是标量，顶点随机值对周围噪声值的权重贡献是按照距离来计算的，不管 0° ~ 360° 的任意方向，相同的距离就有该顶点相同的权重。针对这种情况，便有了大名鼎鼎的 Perlin Noise 柏林噪声。和 Value Noise 类似，都是划分成晶格，噪声点的随机值由点到所在 Cell 的几个顶点的插值生成。区别在于 Perlin Noise 把顶点由伪随机值换成了伪随机向量。向量有方向，所以顶点参与噪声计算中，不同的距离和不同的方向都会产生影响，这样避免了顶点和边界的对称。

  这里要强调下 SCALE_PERLIN 的计算，如果顶点到噪声点的向量和顶点处的随机向量方向一致，即 pointWeight = dot(hash2D, pv) = |hash2D| * |pv| * cos0° = |hash2D| * |pv|。随机向量 hash2D 没有归一化，产生向量范围 [[-1, 1], [-1, 1]]，所以其模最大值为 2\sqrt{2} 2​。又因为后面的插值运算 |pv| 越大，说明离该顶点越远，该点的权重贡献越小。以 1D 来说即 lerp=x+(1−x−x)⋅xlerp = x + (1 - x - x) \cdot xlerp=x+(1−x−x)⋅x，当 x=0.5x = 0.5x=0.5 时取得最大值。2D 就是正方形中心，即 |pv| = 22\frac{\sqrt{2}}{2} 22​​，得到四角顶点权重为 2⋅22=1\sqrt{2} \cdot \frac{\sqrt{2}}{2} = 12​⋅22​​=1。所以 Perlin Noise 后，噪声值刚好为 [-1, 1]，SCALE_PERLIN = 1。如果 hash2D 的向量做了归一化，那么 SCALE_PERLIN 就要再乘以 2\sqrt{2} 2​。两者效果大体相似，做了归一化处理的黑白对比度更高；未做的更“糊”，但性能更好。通常实际应用中需要根据伪随机函数的效果和实际需求再做调整。

Perlin Noise：左边是8×8，右边是16×16

  Perlin Noise 分为几个阶段，从 80 年代提出的 Perlin Noise，到后面再结合 FBM (Fractional Brownian Motion) 分形布朗运动（再结合 Ray Marching 渲染天空云层地面等等，以后再开一篇介绍），直到 2000 年后的改进版结合 Simplex。由于 Perlin Noise 算法的贡献，作者也获得了第 69 届奥斯卡科学技术奖。

Perlin Noise + FBM

  上面的 Perlin Noise 有什么不足吗？就是过渡区域太过于“平滑”了。试想现实中的地形，我国西高东低，呈 3 级阶梯分布。Perlin Noise 中可以完美模拟出地形的高中低三级阶梯式过渡分布，但每级阶梯内部其实也有高高低低错综复杂的地形细节。丢失了这些细节模拟，画面又回到了简单机械呆板。所以在平滑过渡的区域中，继续添加噪声即可增加细节。最简单的做法是根据分形的自相似性，利用原来的噪声函数就行，迭代几次高频低振幅的噪声叠加即可，代码都不用怎么修改。另外分形的思想也可以应用于其他类型的噪声，unity 提供的噪声 shader 中 Simple Noise，就是 Value Noise 结合 FBM，可自行翻阅。Octaves 为迭代次数，一般至少保证 3 次以上，“周期三意味着混沌”。如果想看 FBM 的信号分析，可以看这篇。

Perlin Noise + FBM：左边是4×4，中间是8×8，右边是16×16

8×8 Value Noise + FBM：左边是 1 次循环即默认的 Value Noise，中边是共 3 次，右边是 6 次

Perlin Noise + Simplex

  以上面方式划分晶格时，对于 3 维或更高维空间时，每个晶胞的顶点个数呈几何式暴增。Perlin Noise 在 2001 年便推出了改进版 Simplex。正是借鉴了晶体结构的概念，以最简单的单形填充完整的空间，来计算噪声时减少顶点的参与数量，维度越高减少得越明显。这样首先要解决的是依据噪声所在的 Cell（正方形或正方体），找到对应的单形 Cell，就需要晶格坐标之间的相互变换。下面以 2D 为例，skew/shearing/斜切变换/错切变换（各种各样的叫法）正好可以把原来的正方形 Cell 变换到两个等边三角形组成的菱形。
